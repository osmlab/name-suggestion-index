Wikidata Warnings

The table below shows all the warnings generated when "npm run wikidata" is run.

When the command "npm run wikidata" is run by a maintainer of the project, this will connect to Wikidata directly and attempt to download logos and descriptions for each entry within the Name Suggestion Index (NSI). Sometimes, Wikidata won't be able to provide the data the NSI is looking for, and will return an error message. These error messages are displayed below. Common errors are social media accounts listed in a Wikidata entry which are out of date, or simply incorrect.

Guidance for resolving warnings:

  • Deleted Wikidata entries: If a Wikidata entry has been deleted, it might be possible to request that it is reinstated by the Wikidata administrator who deleted it. Clicking the Wikidata link of the deleted entry will bring up the deletion log for the entry, including the username of the administrator who performed the deletion. For guidance on how to request the undeletion of an entry, please view the "Editing Wikidata" wiki page on the NSI GitHub.
  • Facebook account access ("Unsupported get request"): Some branded Facebook accounts are set with access restrictions, such as requiring the user to be logged in when browsing. Calls to the Facebook API will return a generic error when attempting to read these accounts, which is something the NSI can't skirt around at the moment. If the exact access restriction is identified, it can be added to the "Facebook username" statement on the Wikidata entry using the qualifier "online access status". "npm run wikidata" will ignore the Facebook API warning for any accounts with "online access status" set to "private account", "registration required" (i.e., only visible when logged in), or "location restricted to country of publication" (i.e., the profile is only visible to users from certain countries, or is blocked to users from certain countries). "npm run wikidata" will also warn of any Facebook accounts that have a restricted access qualifier on Wikidata, but have since had their privacy settings changed to make the account publicly viewable.
  • unresolved-redirects: An "unresolved-redirect" is when the Wikidata entry the NSI links to is redirecting to a different (possibly newer) Wikidata entry. In this instance, the Wikidata reference will need to be updated within the NSI data itself.

If you can, simply click on a Wikidata link below, and check to see if the error can be fixed. Please note that fixing an error will not remove the error message below until "npm run wikidata" has been run again, and "npm run dist" has been run to have nsi.guide pull the latest data from the NSI project.
